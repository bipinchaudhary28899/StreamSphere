StreamSphere — Software Architecture

A full-stack video streaming platform (Angular 19 SPA · Express/Node REST API · MongoDB Atlas · Redis · AWS S3 + CloudFront). Four diagrams covering system context, technical architecture, request lifecycle, and the data model.

Diagram 3 · Request / Data Flow

The full lifecycle of an authenticated write request (e.g. liking a video). The request descends through validation and authorization gates; any failed gate returns a structured JSON error (right). On success it reaches the business logic, checks the Redis cache, hits MongoDB on a miss, writes through the cache, and returns 200.

Diagram 4 · Database / Entity Relationship Diagram

User owns Videos, Comments and WatchHistory; Video owns Comments; Comment self-references for threaded replies; WatchHistory is the associative collection between User and Video; likes/dislikes form a many-to-many via embedded arrays on Video.
